# Roadmap внедрения

> **Принцип планирования:** календарные даты не фиксируются до согласования и защиты паспорта в управляющей компании.
> **Точка отсчёта T0:** дата формального решения управляющей компании о запуске проекта.
> Все сроки ниже — относительные интервалы от T0, а не обещание календарных дат.

## Как работает гибкий roadmap

Проект ведётся по модели **stage-gate**:

- следующий этап не запускается автоматически по календарю;
- переход возможен только после достижения результата предыдущего этапа и решения ответственных;
- при задержке согласований, отсутствующих данных или инфраструктурных ограничениях меняется дата старта следующего этапа, но не состав работ;
- конкретный календарный план формируется как rolling-wave план на ближайшие 4–8 недель после T0 и пересматривается еженедельно.

## Нулевая фаза — защита и согласование паспорта

**Период:** до T0, без фиксированной даты завершения.
**Статус:** в работе.

### Gate G0 — решение о запуске

| Решение | Действие |
|---|---|
| Одобрено | Фиксируется T0; запускается этап 1 |
| Одобрено с условиями | Вносится список условий, назначаются владельцы; T0 — после выполнения критичных условий |
| Отправлено на доработку | Паспорт и материалы корректируются; дата защиты назначается отдельно |
| Перенесено | Паспорт остаётся актуальным документом; раз в месяц проводится актуализация предпосылок и рисков |
| Не одобрено | Фиксируются причины и условия возможного возврата к инициативе |

### Gate G2

Можно переходить к приоритизации, когда для сценариев-кандидатов определены владелец процесса, доступность данных и предварительное решение ИБ.

## Этап 3 — приоритизация и дизайн пилотов

**Ориентир:** T0 + 5–8 недель.
**Длительность:** 1–2 недели после G2.

### Результаты

- [ ] Оценён бизнес-эффект и трудозатраты текущего процесса
- [ ] Оценены сложность, интеграции и готовность данных
- [ ] Оценены ИБ- и правовые риски
- [ ] Выбраны 2–3 пилота
- [ ] Для каждого пилота назначены владелец, метрики и критерий успеха
- [ ] Определён целевой контур и способ реализации

### Gate G3 — допуск пилотов

AI-комитет/ответственные утверждают перечень пилотов, бюджет/ресурсы и условия ИБ-допуска.

## Этап 4 — подготовка и запуск пилотов

**Ориентир:** T0 + 7–14 недель.
**Длительность:** 4–6 недель после G3; для интеграционных сценариев может быть больше.

### Результаты

- [ ] Подготовлен тестовый контур
- [ ] Подготовлен разрешённый набор данных
- [ ] Настроены доступы, журналирование и необходимые guardrails
- [ ] Собран MVP / прототип
- [ ] Проведено пользовательское тестирование
- [ ] Собраны фактические метрики «до/после»

## Этап 5 — оценка пилотов и решение о масштабировании

**Ориентир:** T0 + 13–16 недель.
**Длительность:** 1–2 недели после завершения каждого пилота.

### Gate G4 — решение по каждому пилоту

| Решение | Критерий |
|---|---|
| Масштабировать | Достигнут эффект, соблюдены ИБ-требования, есть владелец эксплуатации |
| Доработать | Гипотеза перспективна, но не достигнуты отдельные метрики / есть устранимые ограничения |
| Остановить | Эффект недостаточен, риск неприемлем или данных/владельца нет |

## Этап 6 — масштабирование

**Ориентир:** после G4; отдельная волна планирования.
**Календарные сроки:** определяются только по итогам пилотов.

### Результаты

- [ ] Утверждена целевая архитектура
- [ ] Определён владелец AI-платформы и модель эксплуатации
- [ ] Утверждён governance и регламент использования ИИ
- [ ] Подготовлен план интеграций и развития данных
- [ ] Проведено обучение пользователей
- [ ] Сформирован портфель следующей волны сценариев

## Управление неопределённостью по срокам

| Событие | Реакция в roadmap |
|---|---|
| Защита паспорта переносится | T0 не устанавливается; нулевая фаза продолжается, материалы актуализируются раз в месяц |
| Одобрение с условиями | Создаётся список условий, T0 назначается после закрытия критичных пунктов |
| Недоступны данные | Сценарий не блокирует весь проект: переносится в backlog, выбирается альтернативный пилот |
| Не готова инфраструктура | Запускается пилот на публичных/обезличенных данных параллельно с подготовкой контура |
| Нет владельца процесса | Сценарий не идёт в пилот до назначения владельца |
| Появляется новый приоритет бизнеса | AI-комитет переоценивает реестр, изменения фиксируются в [decisions/Журнал решений](./decisions/%D0%96%D1%83%D1%80%D0%BD%D0%B0%D0%BB%20%D1%80%D0%B5%D1%88%D0%B5%D0%BD%D0%B8%D0%B9.md) |

## Правило коммуникации сроков

Во всех статусах используются две колонки:

| Показатель | Формат |
|---|---|
| Плановый ориентир | Относительно T0: например, «T0 + 2–6 недель» |
| Фактический статус | «Не начат / в работе / ожидает решения Gx / завершён» |

Календарные даты появляются только после Gate G0 и уточняются rolling-wave планом на ближайшие 4–8 недель.
